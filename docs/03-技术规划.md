# 技术规划

> 涵盖技术选型、架构设计、数据库设计、接口契约、项目结构与部署说明。

---

## 一、技术选型

| 层级 | 选型 | 理由 |
|------|------|------|
| **Web 框架** | **FastAPI** | 现代异步框架、原生类型注解、自动 Swagger、代码量少、面试官认可度极高（相对 Django 更适合展示工程化能力） |
| **ORM** | **SQLAlchemy 2.0** (async) 或 **SQLModel** | 与 FastAPI 生态契合，支持异步 |
| **数据库** | **PostgreSQL** | 与 MySQL 任选，PG 对 JSON 字段支持更好（address 字段可直接存 JSONB） |
| **缓存** | **Redis** | Token 存储 + 白名单缓存 |
| **HTTP 客户端** | **httpx** (async) | FastAPI 异步生态标配，比 requests 更适合 |
| **密码加密** | **bcrypt** | 行业标配，绝不可明文存储 |
| **迁移工具** | **Alembic** | 项目当前使用 `Base.metadata.create_all` 自动建表，Alembic 已移除 |
| **配置管理** | **pydantic-settings** + `.env` | 类型安全、环境隔离、面试加分项 |

> 若对 FastAPI 不熟，Django + DRF 也能完成，但代码量会多 30%-50%。

## 二、架构设计

### 2.1 核心流程图

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│   Postman   │────▶│  FastAPI App │────▶│  PostgreSQL │
└─────────────┘     └──────────────┘     └─────────────┘
                           │
                           ▼
                    ┌──────────────┐
                    │    Redis     │
                    │ (Token/白名单)│
                    └──────────────┘
                           │
                           ▼
                    ┌──────────────┐
                    │ meiguodizhi  │
                    │  外部API     │
                    └──────────────┘
```

### 2.2 请求生命周期

1. **请求进入** → CORS/异常中间件
2. **白名单检查** → 若在白名单，直接放行；否则进入鉴权
3. **Token 鉴权** → 从 Header 提取 `Authorization: Bearer <token>`，查 Redis 验证有效性
4. **业务处理** → 调用 Service/CRUD 层
5. **统一响应** → 无论成功/失败，均包装为 `{"code", "data", "msg"}` 格式

## 三、数据库设计

### 3.1 用户表（users）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, Auto Increment | 主键 |
| account | VARCHAR(64) | UNIQUE, NOT NULL, INDEX | 登录账号 |
| pwd | VARCHAR(255) | NOT NULL | bcrypt 哈希，永不存明文 |
| name | VARCHAR(64) | NOT NULL | 用户昵称 |
| created_at | TIMESTAMP | DEFAULT now() | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT now() | 更新时间 |

### 3.2 SQLAlchemy 模型示例

```python
from datetime import datetime
from sqlalchemy import DateTime, Integer, String, func
from sqlalchemy.orm import Mapped, mapped_column

class User(Base):
    __tablename__ = "users"

    id: Mapped[int] = mapped_column(Integer, primary_key=True, autoincrement=True)
    account: Mapped[str] = mapped_column(String(64), unique=True, nullable=False, index=True)
    pwd: Mapped[str] = mapped_column(String(255), nullable=False)
    name: Mapped[str] = mapped_column(String(64), nullable=False)
    created_at: Mapped[datetime] = mapped_column(DateTime, server_default=func.now())
    updated_at: Mapped[datetime] = mapped_column(DateTime, server_default=func.now(), onupdate=func.now())
```

### 3.3 白名单设计建议

- **方案 A（简单）**：代码常量 `WHITELIST_PATHS = ["/api/v1/auth/login", "/docs", "/openapi.json"]`
- **方案 B（进阶）**：数据库 `whitelist` 表 + Redis 缓存，支持动态增删

面试任务建议**方案 A + Redis 缓存**的混合：常量定义 + 启动时写入 Redis Set，兼顾简洁与展示点。

## 四、接口契约设计

### 4.1 统一响应格式

```json
{
  "code": 200,
  "data": {},
  "msg": "success"
}
```

### 4.2 错误码定义

| Code | 含义 | 场景 |
|------|------|------|
| 200 | 成功 | 通用 |
| 400 | 参数错误 | 缺少 account/pwd、格式校验失败 |
| 401 | 未授权 | Token 缺失 / 过期 / 无效 |
| 403 | 禁止访问 | Token 有效但权限不足（如需要可扩展） |
| 500 | 服务端错误 | 外部 API 超时 / 异常、数据库连接失败 |

### 4.3 登录接口

```http
POST /api/v1/auth/login
Content-Type: application/json

{
  "account": "test",
  "pwd": "123456"
}
```

**成功响应**：
```json
{
  "code": 200,
  "data": {
    "token": "a1b2c3d4e5f6...",
    "expires_in": 3600
  },
  "msg": "success"
}
```

**失败响应**：
```json
{
  "code": 400,
  "data": null,
  "msg": "账号或密码错误"
}
```

### 4.4 业务接口（美国地址生成）

```http
GET /api/v1/address/generate
Authorization: Bearer <token>
```

**成功响应**：
```json
{
  "code": 200,
  "data": {
    "address": {
      "Address": "4427 Kincheloe Road",
      "City": "Portland",
      "State": "OR",
      "Zip_Code": "97232",
      "Telephone": "503-527-5486",
      "Full_Name": "Benai",
      ...
    }
  },
  "msg": "success"
}
```

**失败响应**：
```json
{
  "code": 500,
  "data": null,
  "msg": "外部地址服务暂不可用"
}
```

## 七、外部 API 集成策略

### 7.1 正常流程

```python
async def fetch_us_address() -> dict:
    async with httpx.AsyncClient(timeout=10.0) as client:
        resp = await client.post(
            settings.ADDRESS_API_URL,
            headers={"Referer": settings.ADDRESS_API_REFERER},
            json={"path": "/", "method": "address"}
        )
        resp.raise_for_status()
        data = resp.json()
        return data["address"]
```

### 7.2 降级策略（Mock）

若外部 API 不可用（网络波动、反爬升级、服务下线），返回本地 Mock 数据，确保面试官测试时你的服务不会挂掉。

```python
MOCK_ADDRESS = {
    "Address": "1600 Amphitheatre Parkway",
    "City": "Mountain View",
    "State": "CA",
    "State_Full": "California",
    "Zip_Code": "94043",
    "Telephone": "650-253-0000",
    "Full_Name": "Mock User"
}
```

> 面试时主动说明："因外部服务稳定性不可控，已做 Mock 兼容，真实环境切换配置即可。"

## 八、本地运行步骤（供 README 使用）

```bash
# 1. 克隆项目
git clone <repo-url>
cd interview-task

# 2. 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Linux/Mac
# venv\Scripts\activate  # Windows

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置环境变量
cp .env.example .env
# 编辑 .env，填写你的 DB / Redis 连接信息

# 5. 初始化测试用户（自动建表 + seed）
python scripts/init_db.py

# 7. 启动服务
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 8. 访问文档
# Swagger UI: http://localhost:8000/docs
# ReDoc: http://localhost:8000/redoc
```

## 九、测试验证清单

| 场景 | 操作 | 预期结果 |
|------|------|----------|
| 登录成功 | POST `/api/v1/auth/login` 正确账号密码 | 返回 code=200，含 token |
| 登录失败 | POST `/api/v1/auth/login` 错误密码 | 返回 code=400，无 token |
| 无 Token 访问业务 | GET `/api/v1/address/generate` 无 Header | 返回 code=401 |
| 无效 Token | 携带伪造 Token | 返回 code=401 |
| 正常获取地址 | 携带有效 Token | 返回 code=200，data.address 为对象 |
| 白名单绕过 | 访问 `/docs` 无 Token | 正常显示 Swagger |
| Token 过期 | 等待过期时间后访问 | 返回 code=401 |

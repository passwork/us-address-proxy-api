# 项目运行指南

本文档说明如何启动 US Address Proxy Service 并进行接口验证。

---

## 方式一：Docker Compose 一键启动（推荐）

适用于希望零配置快速体验项目的场景。

### 环境准备

- 安装 [Docker Desktop](https://www.docker.com/products/docker-desktop)
- 确保 Docker 和 Docker Compose 命令可用

### 启动步骤

```bash
# 进入项目根目录
cd us-address-proxy-api

# 构建镜像并启动全栈服务
docker-compose up --build
```

首次构建可能需要 2-5 分钟（需下载 PostgreSQL、Redis、Python 基础镜像）。当看到以下日志时，表示服务已就绪：

```
us_address_app  | Created user: test
us_address_app  | INFO:     Application startup complete.
us_address_app  | INFO:     Uvicorn running on http://0.0.0.0:8000
```

### 访问服务

| 地址 | 说明 |
|------|------|
| http://localhost:8000/docs | Swagger UI 接口文档 |
| http://localhost:8000/openapi.json | OpenAPI 规范 |
| http://localhost:8000/api/v1/auth/login | 登录接口 |
| http://localhost:8000/api/v1/address/generate | 地址生成接口 |

### 停止服务

按 `Ctrl + C` 停止，或在另一个终端执行：

```bash
docker-compose down
```

## 方式二：本地 Python 开发启动

适用于需要调试代码或已具备本地开发环境的场景。

### 环境准备

- Python 3.11+
- PostgreSQL 15+ 服务（可用 Docker 快速启动）
- Redis 服务（可用 Docker 快速启动）

> **注意**：项目已删除 SQLite 支持，数据库统一使用 PostgreSQL。本地开发前请确保 PostgreSQL 与 Redis 均已启动。

### 一键脚本

> 执行脚本前，请先启动 PostgreSQL 和 Redis：

```bash
# 启动 PostgreSQL（端口 5433，与项目默认配置一致）
docker run -d --name postgres -p 5433:5432 -e POSTGRES_USER=user -e POSTGRES_PASSWORD=password -e POSTGRES_DB=us_address_proxy postgres:15-alpine

# 启动 Redis
docker run -d --name redis -p 6379:6379 redis:7-alpine
```

然后执行一键脚本：

```bash
# Linux / macOS
bash scripts/setup_venv.sh

# Windows PowerShell
.\scripts\setup_venv.ps1
```

脚本会自动创建虚拟环境、安装依赖、运行测试。完成后手动启动服务即可：

```bash
uvicorn app.main:app --reload --port 8000
```

### 手动步骤

```bash
# 1. 启动 PostgreSQL（如未安装，可用 Docker 快速启动）
# 项目默认连接 localhost:5433，请确保映射到该端口
docker run -d --name postgres -p 5433:5432 -e POSTGRES_USER=user -e POSTGRES_PASSWORD=password -e POSTGRES_DB=us_address_proxy postgres:15-alpine

# 2. 启动 Redis（如未安装，可用 Docker 快速启动）
docker run -d --name redis -p 6379:6379 redis:7-alpine

# 3. 创建虚拟环境并安装依赖
python -m venv venv
source venv/bin/activate        # Linux/Mac
# .\venv\Scripts\activate       # Windows
pip install -r requirements.txt

# 4. 初始化测试账号
python scripts/init_db.py
# 预期输出: Created user: test

# 5. 启动 FastAPI 服务
uvicorn app.main:app --reload --port 8000
```

## Postman 接口验证

### 导入 Collection

1. 打开 Postman 桌面客户端
2. 点击左上角的 **Import** 按钮
3. 选择文件：`docs/US-Address-Proxy-API.postman_collection.json`
4. 导入成功后，左侧集合列表会出现 **US Address Proxy API**

### 执行测试

按照以下顺序逐个发送请求：

#### Step 1：登录获取 Token

- 请求: `POST /api/v1/auth/login`
- Body (raw JSON):
  ```json
  {"account": "test", "pwd": "123456"}
  ```
- 预期响应:
  ```json
  {"code": 200, "data": {"token": "...", "expires_in": 3600}, "msg": "success"}
  ```

#### Step 2：携带 Token 获取地址

- 请求: `GET /api/v1/address/generate`
- Headers:
  ```
  Authorization: Bearer <Step1 获取的 token>
  ```
- 预期响应:
  ```json
  {"code": 200, "data": {"address": {"Address": "...", "City": "..."}}, "msg": "success"}
  ```

#### Step 3：无 Token 访问地址（鉴权拦截）

- 请求: `GET /api/v1/address/generate`
- Headers: 留空
- 预期响应:
  ```json
  {"code": 401, "data": null, "msg": "未授权，缺少有效Token"}
  ```

#### 附加测试

Collection 中还包含以下用例，可逐个点击 **Send** 验证：

| 用例名 | 预期状态码 | 预期 code | 说明 |
|--------|-----------|-----------|------|
| 错误密码登录 | 400 | 400 | 密码错误返回统一错误 |
| 缺少密码 | 422 | 422 | Pydantic 参数校验 |
| 白名单 /docs | 200 | — | 免鉴权访问 |

### 自动化 Runner（可选）

1. 点击集合名称右侧的 **Run** 按钮
2. 勾选全部 6 个请求
3. 点击 **Run US Address Proxy API**
4. 预期结果: `6/6 passed`

## 常见问题

### Q: 启动时提示 Redis 连接失败？

**A**: 方式一（Docker Compose）已内置 Redis，通常不会出现此问题。若使用方式二，请确保 Redis 已启动且端口 6379 未被占用。

### Q: `docker-compose up --build` 卡在构建阶段？

**A**: 首次构建需要下载 PostgreSQL、Redis、Python 基础镜像，请耐心等待 2-5 分钟。如网络较慢，可尝试更换 Docker 镜像源。

### Q: 容器启动后 app 服务报错无法连接 PostgreSQL？

**A**: 可能是 PostgreSQL 尚未完全初始化。`docker-compose.yml` 中已配置 `sleep 3` 等待，如仍失败请尝试：

```bash
docker-compose down -v
docker-compose up --build
```

### Q: 登录接口返回 "账号或密码错误"？

**A**: 测试账号未成功初始化。Docker Compose 方式下，`init_db.py` 会在容器启动时自动执行。如失败，可手动执行：

```bash
docker-compose exec app python scripts/init_db.py
```

### Q: 本地开发时 pytest 报错无法连接 PostgreSQL？

**A**: 本地开发依赖外部 PostgreSQL 服务。请确保已通过 Docker 或本地安装启动 PostgreSQL，且端口与 `app/config.py` 中的 `database_url` 一致（默认为 `localhost:5433`）。如需修改端口或数据库地址，可创建 `.env` 文件覆盖：

```bash
# .env
DATABASE_URL=postgresql+asyncpg://user:password@localhost:5432/us_address_proxy
```

### Q: 如何修改外部 API 超时时间或降级策略？

**A**: 编辑 `app/config.py` 中的 `address_api_timeout` 和 `address_api_mock_on_failure`，或创建 `.env` 文件覆盖配置。
